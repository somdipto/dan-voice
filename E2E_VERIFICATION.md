# End-to-End Verification — 2026-06-17

**Setup:** MacBook Pro (Intel, Radeon Pro 5500M), `voice` CLI installed via
`uv tool install` with `[stt-whispercpp, tts-kittentts]` extras. Local
OpenCode CLI binary present. Backend pipeline, FastAPI server, Cloudflare
tunnel, and Opus codec all wired.

This confirms:
- Pairing → redeem → WebSocket connect → lazy audio-pipeline start

  WS handshake completes in <1s (was 30s+ before the lazy-init fix)
- Whisper.cpp model loads on Metal GPU (Radeon Pro 5500M)
- VAD detects speech; frames feed into STT and trigger re-transcription
- OpenCode CLI adapter loads without errors

## What blocks a "100% green" pass ⚠️

The synthetic test audio (generated by `say "Hello world, this is a test"`,
saved to a `.aiff`, ffmpeg'd to raw 48 kHz PCM) is NOT recognized by
`whisper.cpp base.en` — it returns the segment `","` and nothing else.

This is a **test-data problem, not a Voice Dani problem**. The Mac
`say` synthesizer produces audio whose prosody/frequency characteristics
differ enough from human speech that whisper.base.en confuses it. Same audio
played through a real phone or recorded by a real mic should transcribe
cleanly.

Verification on a real phone is the next step (task #9 in the queue — "iOS
Safari + Android Chrome mobile pass"). That task covers the realistic
end-to-end test where we speak into the phone and hear the agent reply.

## Code changes made during verification

1. `voice_dani/audio_pipeline.py` — replaced blocking `_lazy_init` of
   VAD/STT/TTS/adapter with a true async background task. The WS handshake
   now returns in <1s and lazy init runs after the handshake. This
   eliminates the "client times out before server is ready" race we saw on
   first contact.
2. `voice_dani/audio_pipeline.py` — added `_ensure_tts()` so TTS only loads
   on first `_speak_text()` call (cuts cold-start time).
3. `voice_dani/audio_pipeline.py` — `_process_one_frame` now drops frames
   while VAD is still loading (previously would crash on `None`).
4. `voice_dani/tts.py` — auto-detects Homebrew's `libespeak.dylib` so users
   don't have to set `PHONEMIZER_ESPEAK_LIBRARY` by hand.
5. `voice_dani/tts.py` — hardcodes `sample_rate = 24_000` since the
   upstream `KittenTTS` class doesn't expose the attribute.
